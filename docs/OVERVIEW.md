# Hidden Wallet - Обзор

## Что такое Hidden Wallet?

Hidden Wallet — это продвинутый слой безопасности для Solana кошельков, который обеспечивает полную приватность и защиту ваших транзакций. Проект вдохновлен концепцией CloakZK и предоставляет многоуровневую систему защиты, скрывающую ваш основной кошелек от анализа блокчейна.

## Основные концепции

### Скрытый слой (Hidden Layer)

Hidden Wallet работает как промежуточный слой между вашим основным Solana кошельком и блокчейном. Все транзакции проходят через этот слой, который:

- **Скрывает источник транзакций**: Ваш реальный адрес кошелька остается скрытым
- **Обфусцирует метаданные**: Защищает от анализа временных паттернов и объемов
- **Маршрутизирует через прокси**: Использует децентрализованную сеть для анонимизации

## Ключевые возможности

### 1. Многоуровневое шифрование

- **AES-256-GCM**: Для шифрования приватных ключей
- **ChaCha20-Poly1305**: Для потокового шифрования
- **RSA-OAEP**: Для асимметричного шифрования

### 2. Приватность транзакций

- **Смешивание адресов**: Разрыв связи между отправителем и получателем
- **Транзакционное смешивание**: Объединение транзакций для приватности
- **Обфускация метаданных**: Защита от временного и объемного анализа

### 3. Децентрализованная маршрутизация

- **Прокси-сеть**: Распределенная сеть узлов для маршрутизации
- **Балансировка нагрузки**: Автоматический выбор оптимальных узлов
- **Отказоустойчивость**: Автоматическое переключение при сбоях

### 4. Уровни безопасности

#### Базовый (Basic)
- Шифрование ключей AES-256
- Базовая обфускация адресов
- Прямые транзакции

#### Стандартный (Standard)
- Расширенное шифрование
- Обфускация трафика
- Смешивание через один промежуточный адрес

#### Высокий (High)
- Полное шифрование AES-256-GCM
- Многоуровневая обфускация
- Децентрализованная маршрутизация
- Многоуровневое смешивание транзакций

## Как это работает

### Процесс отправки транзакции

1. **Инициализация**: Пользователь подключает свой Solana кошелек
2. **Создание скрытого адреса**: Генерируется промежуточный адрес
3. **Шифрование**: Приватные ключи шифруются локально
4. **Смешивание**: Транзакция маршрутизируется через промежуточные адреса
5. **Прокси**: Запрос проходит через децентрализованную сеть прокси
6. **Отправка**: Транзакция отправляется в блокчейн

### Защита от анализа

Hidden Wallet защищает от следующих типов анализа:

- **Графовый анализ**: Разрыв связей между адресами
- **Временной анализ**: Случайные задержки и вариации
- **Объемный анализ**: Разделение транзакций на части
- **Корреляционный анализ**: Обфускация паттернов

## Использование

### Быстрый старт

```typescript
import { HiddenWallet } from '@hidden-wallet/core';

const wallet = new HiddenWallet({
  network: 'mainnet-beta',
  securityLevel: 'standard'
});

await wallet.connect();
```

### Отправка приватной транзакции

```typescript
const signature = await wallet.sendTransaction({
  to: 'destination_address',
  amount: 1.5,
  hideOrigin: true
});
```
